# 常见问题

提示

提问之前，请先查阅下面的常见问题。


# 如何使用 MapGIS Pan-Spatial Map ？

请阅读文档 开始使用，有任何疑问，欢迎在 github 上给我们提交 issue (opens new window)。

# 首先有任何报错，最简单的方法是把报错信息复制到浏览器里面搜索一下！！！


# 代码下载慢？

npm 或者 github 有时候因为中国墙的原因，网速会不稳定。有如下三种解决方案：

  • 使用淘宝源

    npm install --registry=https://registry.npm.taobao.org

  • Yarn

    • yarn 介绍 (opens new window)
    • 安装 (opens new window)

    yarn install

  • 自己科学上网

# vendor 过大问题?

建议使用 gzip，使用之后体积会只有原先 1/3 左右。还可以使用懒加载或者 Code Splitting 建议参考这篇文章 (opens new window)。打出来的 app.js 过大，查看一下是不是 Uglify 配置不正确或者 sourceMap 没弄对。 优化相关请看该文章 (opens new window)。


# 我的 url 里怎么有 # 号？要如何去掉？

请参考文档 前端路由与服务端的结合 主要由于 history 和 hash 两种不同模式导致的。


# npm install 总是失败？

由于中国墙的的原因，安装一些依赖时很容易出现超时问题，国内用户推荐使用淘宝源的安装方式

npm install --registry=https://registry.npm.taobao.org

# node-sass 安装失败？

建议参考这里 (opens new window)或者这里 (opens new window)

# mock 问题?

请参考文档 Mock Data


# 图片 background url() 引用问题？

https://github.com/vuejs/vue-loader/issues/481 (opens new window)

# can't not find 'xxModule' - 找不到某些依赖或者模块?

这种情况一般报错信息可以看到是哪个包抛出的信息. 一般卸载这个模块,安装重新安装下即可.

# JavaScript heap out of memory?

这种情况解决办法可以是增加 nodejs 可使用的最大内存大小，也可以从降低程序内存消耗的角度入手。


# 其它任何关于 vue 的问题 ？

请你首先仔细阅读 vue 官方文档 (opens new window),但部分问题其实看报错就能知道原因了！


更多常见问题可以查看 pro/issue (opens new window) 是否能解决。如果这里未能解决你的问题，欢迎 报告给我们 (opens new window)。
